When events happen, they're "captuted" from root node (top-down). But events are NOT triggered yet.
By default (bubbling), after event capture are done, browser starts firing the events for nodes with event listener, from bottom-up.
Only when setting `{capture:true}` to event listener, events will be fired top-down.
`once` is a new attibute to unbind listener automatically after the callback is execute once.
